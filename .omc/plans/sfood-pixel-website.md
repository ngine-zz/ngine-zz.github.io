## 1. 개요

메인 웹사이트에서 진입하는 **픽셀아트 인터랙티브 경험**입니다.  
구역 구조·상호작용 흐름은 [`ia-pixel-village.md`](ia-pixel-village.md)에서 관리하며, 이 문서는 기술 구현 기획에 집중합니다.

### 핵심 가치

| 가치 | 설명 |
|---|---|
| **발견의 재미** | 예상치 못한 오브젝트/NPC와의 상호작용 (이스터에그) |
| **몰입감** | 인게임 컨버세이션 방식으로 콘텐츠를 전달해 끊김 없는 경험 |
| **접근성** | 데스크탑(키보드) + 모바일(가상 조이스틱) 모두 지원 |
| **글로벌** | 한국어/영어 전환 지원 |

## 2. 타겟 대상

- **고객**: 에쓰푸드 제품과 브랜드를 탐색하는 일반 소비자
- **투자자**: 회사 개요 및 IR 정보를 확인하는 투자자
- **구직자**: 채용 정보와 회사 문화를 탐색하는 잠재 입사 지원자

---

## 3. 콘텐츠 구역 (4개 핵심 영역)

픽셀아트 세계는 4개의 주요 구역으로 나뉩니다. 각 구역은 별도의 인터랙션 존으로 구성됩니다.

| 구역 | 내용 | 주요 인터랙션 오브젝트 예시 |
|---|---|---|
| **회사 소개/역사관** | 에쓰푸드의 창립 역사, 비전, 미션 | 타임라인 간판, 역사 NPC |
| **제품/브랜드관** | 주요 제품군 및 브랜드 소개 | 제품 오브젝트 (소시지, 스팸, 별미당 등) |
| **조직/팀관** | 부서별·팀별 소개 | 직원 NPC (팀별 구분) |
| **채용/IR관** | 채용 공고, 투자자 정보 | 채용 안내판, IR 서류 오브젝트 |

## 5. 맵 사이즈

### 픽셀 단위 (확정)

| 항목 | 값 | 비고 |
|---|---|---|
| 맵 타일 크기 | **32×32px** | Tiled 배경 타일 기준 |
| 스프라이트 소스 크기 | **128×128px** | PixelLab 생성 기준 |
| Phaser 표시 스케일 | `setScale(0.25)` | 화면상 32×32px로 표시 |

> 스프라이트는 128×128 고해상도 원본을 유지하고 Phaser에서 축소 표시합니다.  
> 품질 손실 없이 표시 크기만 조정하는 방식입니다.

### 맵 크기 (권장)

**100×80 타일 (3,200×2,560px)** 을 기본으로 합니다.

```
┌──────────────────┬──────────────────┐
│   역사/소개관     │   제품/브랜드관   │
│    40×30 타일    │    40×30 타일    │
├──────────────────┼──────────────────┤
│   조직/팀관      │   채용/IR관       │
│    40×30 타일    │    40×30 타일    │
└──────────────────┴──────────────────┘
  구역(40) + 통로(10) + 구역(40) = 90 → 100타일 (여백 포함)
  구역(30) + 통로(10) + 구역(30) = 70 → 80타일  (여백 포함)
```

| 항목 | 값 |
|---|---|
| 맵 크기 | 100×80 타일 |
| 픽셀 크기 | 3,200×2,560px |
| 화면 대비 (FHD) | 가로 2.5배 / 세로 2.3배 |
| 화면에 보이는 타일 수 | 약 60×34 타일 |

### 크기 비교

| 사이즈 | 픽셀 | 평가 |
|---|---|---|
| 60×50 | 1,920×1,600px | 4구역 배치 빡빡함 |
| **100×80** | **3,200×2,560px** | **✅ 권장** |
| 150×120 | 4,800×3,840px | 채울 콘텐츠가 많아야 함 |

## 6. 캐릭터 시스템

### 5-1. 플레이어 캐릭터

- **컨셉**: 에쓰푸드 마스코트 (고정 캐릭터 1종)
- **조작**:
  - 데스크탑: `WASD` + 방향키 이동
  - 모바일: 가상 조이스틱 (Virtual Joystick)
- **상호작용**: 인터랙션 존 진입 후 `Enter` 키 (또는 모바일 탭)

### 5-2. NPC / 오브젝트

자유롭게 또는 랜덤하게 이동하는 요소들로 세계를 채워 생동감을 부여합니다.

| 유형 | 예시 | 상호작용 여부 |
|---|---|---|
| 직원 NPC | 팀별 픽셀아트 직원 캐릭터 | ✅ 대화 가능 |
| 동물 | 소, 오리, 돼지 (식품 연관) | ✅ 대화/반응 가능 |
| 제품 오브젝트 | 소시지, 햄 캔, 별미당 케이크 | ✅ 터치 시 제품 정보 |
| 환경 오브젝트 | 간판, 나무, 자동차 | ✅ 일부 상호작용 |
| 장식 요소 | 새, 구름, 조명 효과 | ❌ 배경 연출용 |

## 7. 상호작용 레이어 시스템

인터랙션의 성격에 따라 3가지 레이어 타입을 사용합니다.

### Type A — 텍스트 박스 (NPC 대화)
- 화면 하단에 고정되는 RPG 스타일 대화창
- 짧은 정보, 유머, 힌트 전달용
- `Enter` / 탭으로 다음 문장 진행

### Type B — 슬라이드 레이어 (상세 정보)
- 화면의 50%를 차지하는 오버레이 레이어
- 긴 텍스트 + 이미지 조합
- 예: 제품 상세 소개, 팀 소개, 채용 공고

### Type C — 풀 레이어 (영상/몰입형 콘텐츠)
- 영상 임베드 (YouTube / 자체 영상)
- 브랜드 필름, 제품 광고, 회사 소개 영상 등

> 모든 레이어는 Angular 컴포넌트로 구현하고, Phaser 씬에서 Angular 서비스를 통해 트리거합니다.

## 8. 다국어 지원

- 지원 언어: **한국어** / **영어**

### 감지 방식

브라우저 내장 언어 설정(`navigator.language`)으로 감지합니다. 외부 API 호출 없이 즉시 감지됩니다.

| 감지 결과 | 기본 제안 언어 |
|---|---|
| `ko`, `ko-KR` | 한국어 |
| 그 외 | 영어 |

### 최초 접속 시 UX 흐름

```
접속
  ↓
navigator.language 감지 (코드 1줄, 비용 0원)
  ↓
게임 시작 전 모달 팝업 표시
  ┌─────────────────────────────┐
  │  한국어로 계속하시겠습니까?  │
  │                             │
  │  [한국어로 계속]  [English]  │
  └─────────────────────────────┘
  ↓
선택한 언어로 게임 시작
  ↓
선택값 localStorage에 저장 (재방문 시 팝업 생략)
```

### 재방문 시
- `localStorage`에 저장된 값이 있으면 팝업 없이 바로 해당 언어로 시작
- 인게임 내 언어 변경을 위한 미니멀 토글 버튼 별도 제공

### 구현
- 언어 감지: `navigator.language` (외부 의존성 없음)
- 언어 적용: `ngx-translate` (런타임 전환 지원)
- 언어 저장: `localStorage('sfood-lang')`

## 9. 기술 아키텍처 (초안)

```
Angular (라우팅 / UI 레이어 / i18n)
  └── GameComponent (Phaser 4 호스팅)
        ├── BootScene (로딩 / 에셋 사전 로드)
        ├── MainScene (월드 탐험 메인 씬)
        │     ├── PlayerController (WASD + 조이스틱)
        │     ├── NPCManager (랜덤 이동 / 대화)
        │     ├── InteractionZoneManager (Enter 감지)
        │     └── ChunkLoader (맵 동적 로드)
        └── UIScene (HUD — 조이스틱, 언어 토글, 인터랙션 힌트)

Angular Overlay Layer (Phaser 위에 absolute 포지션)
  ├── DialogBoxComponent   (Type A)
  ├── SlideLayerComponent  (Type B)
  └── VideoLayerComponent  (Type C)
```

### 주요 기술 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 맵 포맷 | Tiled JSON (`.tmj`) | NPC 스폰, 인터랙션 존, 충돌 레이어 시각적 편집 |
| NPC 이동 | Phaser 트윈 + 랜덤 웨이포인트 | 자연스러운 배회 연출 |
| UI 레이어 | Angular 컴포넌트 (Phaser 위 absolute) | 복잡한 HTML/CSS UI를 Canvas 밖에서 처리 |
| 모바일 조작 | rexUI 가상 조이스틱 또는 커스텀 | Phaser 4 호환 필요 확인 후 선택 |
| 다국어 | ngx-translate | 런타임 전환 지원 |
