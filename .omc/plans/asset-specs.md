## 1. 공통 규격

| 항목 | 값 | 비고 |
|---|---|---|
| **스프라이트 소스 크기** | **128×128px** | PixelLab 생성 크기 (확정) |
| **맵 타일 크기** | **32×32px** | Tiled 맵 배경 타일 기준 |
| **Phaser 표시 스케일** | `setScale(0.25)` | 128×128 → 화면상 32×32px 표시 |
| 색상 팔레트 | TBD | 레퍼런스 이미지 수령 후 확정 |
| 배경 | 투명 (Alpha) | 필수 |
| 포맷 (원본) | GIF (애니메이션) / PNG (정적) | PixelLab 출력 기준 |
| 포맷 (변환) | PNG 스프라이트시트 + JSON | Phaser 로딩용 |
| 뷰 타입 | 탑다운(Top-down) | 레퍼런스: peteroravec.com |

> 스프라이트 소스는 128×128px 고해상도로 유지하고, Phaser에서 `setScale(0.25)`로 축소 표시합니다.  
> 파일 품질 손실 없이 표시 크기만 조정합니다.

## 3. NPC — 직원 캐릭터

**우선순위**: P1

> 직원 NPC는 4방향(N/S/E/W)으로 충분합니다.  
> 대각선은 Phaser에서 가장 가까운 방향으로 대체합니다.

| 종류 | 액션 | 방향 | 프레임 수 | 파일 수 (1종 기준) |
|---|---|---|---|---|
| 직원 NPC | `idle` | 4방향 | 4f | 4 |
| 직원 NPC | `walk` | 4방향 | 6f | 4 |
| **합계 (1종)** | | | | **8개** |

**계획 종류**: 최소 3종 (팀/부서 구분)
- `employee_01`: IT/AX팀 대표 NPC
- `employee_02`: 영업/마케팅 대표 NPC
- `employee_03`: 생산/물류 대표 NPC

**총 파일 수**: 8개 × 3종 = **24개**

## 4. NPC — 동물 (이스터에그 / 세계관 연출)

**우선순위**: P2

> 동물은 좌/우(L/R)만 생성하고, Phaser `flipX`로 반전 처리합니다.

| 종류 | 액션 | 방향 | 프레임 수 | 파일 수 | 소스 크기 | Phaser 표시 스케일 | 화면상 크기 |
|---|---|---|---|---|---|---|---|
| 소 (Cow) | `idle` | L/R | 4f | 2 | 128×128 | `setScale(0.25)` | 32×32px |
| 소 (Cow) | `walk` | L/R | 6f | 2 | 128×128 | `setScale(0.25)` | 32×32px |
| 돼지 (Pig) | `idle` | L/R | 4f | 2 | 128×128 | `setScale(0.18)` | 23×23px |
| 돼지 (Pig) | `walk` | L/R | 6f | 2 | 128×128 | `setScale(0.18)` | 23×23px |
| 오리 (Duck) | `idle` | L/R | 4f | 2 | 128×128 | `setScale(0.12)` | 15×15px |
| 오리 (Duck) | `walk` | L/R | 6f | 2 | 128×128 | `setScale(0.12)` | 15×15px |
| **합계** | | | | **12개** | | | |

> 소스 파일은 모두 128×128px 동일. `setScale` 값만 다르게 줘서 상대적 크기를 표현합니다.  
> 스케일 값은 시각적 테스트 후 미세 조정 가능합니다.

## 9. 에셋 생성 우선순위 로드맵

```
P0 — MVP (첫 번째 프로토타입)
  ├── 마스코트 idle 8방향 + walk 8방향
  ├── 타일셋 기본 1종 (바닥 + 건물)
  └── 대화창 UI (Type A)

P1 — 베타
  ├── 직원 NPC 3종 × 4방향
  ├── 인터랙션 Type B/C 레이어 UI
  └── 가상 조이스틱

P2 — 완성
  ├── 동물 NPC 3종
  ├── 제품 오브젝트
  ├── 추가 타일셋 (deco, UI)
  └── 오디오
```
