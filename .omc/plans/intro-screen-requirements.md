> 이 문서만 수정해 현재 웹사이트 인트로 화면의 기획 요구사항을 변경합니다.

> 상태: 기획 정리 중

## 1. 개요

인트로 화면은 SFood 웹사이트에서 활용되는 별도 페이지로, 기ㄴ SFOOD 공식 웹사이트와는 디자인 스타일이나 표현방법이 전혀 별개여서 독립된 페이지처럼 제작한다. 해당 페이지는 인트로 화면을 가지고 있으며, 인트로 페이지는 뒤에 구현될 여러 에셋들과 소스들을 미리 로딩하는 역할을 수행하는 첫 진입 화면이다. 방문자에게 픽셀아트 기반의 브랜드 월드 분위기를 먼저 전달하고, 이후 Pixel Village 경험으로 이어질 수 있는 관문 역할을 한다.

현재 구현 대상은 `src/app/intro/intro.component`이며, 라우팅상 `/` 경로에 배치된다.

|---|---|

## 5. 사용자 흐름

### 5-1. 기본 흐름

```
사용자 접속 `/`
  ↓
인트로 픽셀아트 배경 표시
  ↓
브랜드명 / 보조 문구 / 진입 CTA 표시
  ↓
사용자 선택
  ├─ Pixel Village 입장
  └─ 일반 메인 웹사이트 콘텐츠 탐색
```

### 5-2. Pixel Village 흐름

```
인트로 화면
  ↓ [Pixel Village 입장하기]
언어 선택 모달
  ↓
로딩 화면
  ↓
Pixel Village MainScene
```

언어 선택 모달과 로딩 화면은 Pixel Village 범위이지만, 인트로 CTA는 해당 흐름을 시작할 수 있어야 한다.

## 6. 콘텐츠 요구사항

| 요소 | 상태 | 요구사항 |
|---|---|---|
| 브랜드명 | 필요 | `SFOOD` 또는 확정 브랜드 표기 사용 |
| 보조 문구 | 필요 | `Better Food, Better World` 또는 확정 슬로건 사용 |
| Pixel Village CTA | 필요 | `/village` 또는 언어 선택 모달로 진입 |
| 메인 웹사이트 CTA | 미정 | 일반 홈 콘텐츠가 생기면 스크롤/섹션 이동 제공 |
| 언어 선택 | 미정 | GNB 또는 모달 정책 확정 후 반영 |
| 푸터/법적 정보 | 제외 | 인트로 단독 화면에는 노출하지 않음 |

## 7. 시각 요구사항

| 영역 | 요구사항 |
|---|---|
| 전체 분위기 | 픽셀아트, 목장/산/구름 기반의 따뜻한 브랜드 월드 |
| 하늘 | 시간대별 색상 변경이 가능한 단색 기반 하늘 |
| 구름 | 지형 뒤로 흐르는 다층 구름 애니메이션. 지형 원본 좌표계와 같은 화면 가로폭 비율을 따라 크기와 위치를 조정 |
| 지형 | 원본 픽셀아트 비율을 유지하며 화면 하단에 고정 |
| 풀/노란꽃 | 노란 풀꽃은 정적 PNG 베이스로 원본 형태를 유지한다. 흔들림은 하단 전경의 일부 작은 SVG 스프라이트 조각에만 적용한다. 노란 풀꽃 전경선은 좌우가 높고 중앙이 낮으므로 단일 y값이 아니라 x 위치별 곡선형 하한선으로 후보를 분할하며, 언덕/중경의 노란 지형·풀꽃 영역은 움직이지 않는다. |
| 흰꽃 | 스크린샷 `2026-06-19 14.17.05`, `2026-06-19 14.16.59`에서 지정된 하단 전경 흰꽃 군집 중 좌우 가장자리 군집을 줄기/잎 정적 베이스 PNG와 흰 꽃잎/꽃술 SVG 스프라이트로 분리한다. 여기서 흰꽃은 흰/크림 꽃잎과 해당 꽃에 붙은 짙은 녹색/연두색 줄기와 잎 묶음을 의미한다. 중앙의 노란 풀꽃과 잔디 사이 영역은 원본 지형을 유지하고, 별도 노란꽃 계열과 어두운 배경색 계열은 흰꽃 베이스에 포함하지 않는다. 분리 후 지형 배경은 투명 구멍이나 단색 패치로 두지 않고 주변 잔디 패턴을 샘플링해 메운다. 꽃잎/꽃술 일부만 독립적으로 잔잔하게 흔들림 |
| 흰꽃 반응형 | 흰꽃은 별도 수평 벌어짐/독립 이동을 적용하지 않고 지형과 같은 원본 비율 좌표계로만 스케일 |
| 레이어 독립성 | 하늘, 구름, 지형, 날씨는 독립 레이어로 유지하되 흰꽃은 지형 비율과 위치 기준을 따른다 |
| 날씨 | 비, 눈, 번개 효과를 조건부로 표시 가능 |
| 콘텐츠 레이어 | 배경을 가리지 않되 첫 화면에서 브랜드/CTA가 즉시 인지되어야 함 |

## 13. 수용 기준

MVP 기준:

1. `/` 접속 시 인트로 배경이 깨지지 않고 전체 뷰포트를 채운다.
2. 브랜드명, 보조 문구, Pixel Village CTA가 첫 화면에서 보인다.
3. CTA를 키보드와 포인터로 실행할 수 있다.
4. 모바일 너비에서도 텍스트와 버튼이 겹치거나 넘치지 않는다.
5. 배경 애니메이션이 CTA 클릭/탭을 방해하지 않는다.
6. `/village` 또는 언어 선택 모달로 이어지는 진입 경로가 존재한다.
7. 기본 메타 타이틀과 문서 언어가 임시값 `Website` / 무조건 `en` 상태로 남지 않는다.

완성 기준:

1. 시간대, 날씨, 계절 상태가 정책에 따라 동작한다.
2. `prefers-reduced-motion` 대응이 적용된다.
3. 한국어/영어 정책과 CTA 문구가 확정되어 반영된다.
4. 일반 메인 웹사이트 홈 구조와 Pixel Village 진입 구조가 충돌하지 않는다.
