## 2. 레이어 구조

Tiled에서 아래 순서로 레이어를 생성한다. (위→아래 = 렌더 위→아래)

| # | 레이어명 | 타입 | 설명 |
|---|---|---|---|
| 1 | `collision` | Object | 충돌 영역 (Phaser Physics) |
| 2 | `interaction-zones` | Object | 인터랙션 트리거 영역 |
| 3 | `spawn-points` | Object | 플레이어 스타트, NPC 스폰 |
| 4 | `zone-labels` | Object | 구역 메타 (이름, 타입 프로퍼티) |
| 5 | `decor-high` | Tile | 나무, 건물 지붕, 높은 오브젝트 |
| 6 | `buildings` | Tile | 건물 본체 (충돌 포함) |
| 7 | `decor-low` | Tile | 울타리, 꽃, 낮은 장식 |
| 8 | `paths` | Tile | 흙길, 포장길 |
| 9 | `ground` | Tile | 잔디, 흙, 물 (기본 지형) |
| 10 | `bg` | Tile | 하늘, 산, 원경 배경 |

> **Object 레이어 프로퍼티 규칙**
> - `zone-labels`: 각 폴리곤에 `zoneId`, `brandKey`, `interactionType(A/B/C)` 커스텀 프로퍼티 부여
> - `interaction-zones`: 각 직사각형에 `zoneId`, `action` 프로퍼티 부여
> - `spawn-points`: 각 포인트에 `type(player/npc)`, `npcId`, `movePattern` 프로퍼티 부여

## 7. 충돌 영역 원칙

1. `collision` Object 레이어에 Rectangle로 배치
2. **건물 본체**: 건물 타일 전체 영역
3. **울타리**: 각 구역 경계 (2타일 두께)
4. **연못**: 물 타일 전체 (진입 불가)
5. **산**: Row 1 상단 산 영역 (진입 불가)
6. **길**: 충돌 없음 (통행 가능)
7. 맵 외곽 4방향: 보이지 않는 충돌 벽 (타일 0, 99, 0, 79)

## 8. Tiled 작업 순서

```
1. 새 맵 생성 → 100×80 타일, 32×32px, Orthogonal  → 검증: 맵 크기 확인
2. 타일셋 임포트 → ground.png, buildings.png 등    → 검증: 타일셋 표시 확인
3. `bg` 레이어 → 하늘·산 배경 페인팅               → 검증: 시각 확인
4. `ground` 레이어 → 잔디·흙 기본 지형             → 검증: 전체 커버리지
5. `paths` 레이어 → 길 네트워크 (3-6 기준)          → 검증: 좌표 일치
6. `decor-low` → 울타리, 꽃, 낮은 오브젝트          → 검증: 구역 경계
7. `buildings` → 건물 배치 (구역별 좌표 기준)        → 검증: 위치 확인
8. `decor-high` → 나무, 지붕                        → 검증: 시각 확인
9. `collision` Object 레이어 → 충돌 영역 설정        → 검증: Phaser Physics 테스트
10. `interaction-zones` → 인터랙션 존 배치           → 검증: Enter 키 트리거 확인
11. `spawn-points` → 플레이어·NPC 스폰 포인트         → 검증: 스폰 동작 확인
12. `zone-labels` → 구역 메타 프로퍼티               → 검증: Phaser에서 읽기 확인
```
